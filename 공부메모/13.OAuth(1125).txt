서드파티에 계정 관리를 맡기는 것이 OAuth (구글 / 네이버 로그인 등)
토큰을 발급해 주는 역할을 하는 어플리케이션,
오너의 정보를 보호하는 주체 (네이버, 구글 등) : 리소스 서버

★(권한 부여 코드 승인) 권한에 접근할 수 있는 코드와 리소스 오너에 대한 액세스 토큰을 발급받는다.
(암시적 승인 타입) 요청 보낼 시 사용자가 인증 과정만을 거치고 액세스 토큰을 발급받는다.
(리소스 소유자 암호 자격증명) 클라이언트의 패스워드 이용, 액세스 토큰에 대한 자격 증명 교환

권한 부여 코드 승인 순서
1) 리소스 오너(사용자) 가 애플리케이션에 권한 요청 - 2) 리소스 서버가 유저에게 데이터 접근용 권한 부여
3) 인증 서버로 인증 코드 발급 -> 4) 애플리케이션으로 액세스 토큰 발급.

요청 URI에는 클라이언트ID /  리다이렉트될 URL / 응답될 타입등을 파라미터로 전송.
response_type? 클라이언트가 제공받는 응답 타입/
로그인 성공 시 redirect_uri의 값으로 리다이렉트됨.
clint_scret : OAuth 서비스 등록 시 제공받는 비밀키.

쿠키 - 유저의 로컬 환경에 저장되는 데이터. (전에 사이트 방문한 적이 있는지, 로그인 정보 등) - 키/값.

----

구글 로그인을 위해서는 인증 서버에서 토큰을 제공받아야 한다.
