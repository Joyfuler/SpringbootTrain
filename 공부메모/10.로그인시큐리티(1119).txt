Spring Security를 활용, 간단한 어노테이션으로 세션 고정 / CSRF 공격 등을 막아낼 수 있음.

회원 등의 데이터를 담은 dto 클래스에 UserDetails 인터페이스를 implements하여 
로그인 상태 / 패스워드 / 계정 사용 여부 등을 체크 가능.

Config 클래스를 만들어서 어떤 경로에 시큐리티를 설정할지, 설정하지 않을 곳은 어딘지,
경로는 어디로 이동할지 등을 설정 가능.

requestMatchers("경로") --> 해당 경로로 이동하는 경우의 설정을 진행.
permitAll() --> 해당 경로로 이동할 때는 시큐리티 설정 없이 모두가 접근 가능. (login / 회원가입 등)

인증 관리자 관련 설정.. 사용자 정보를 가져올 서비스 클래스 세팅 후, 비밀번호 암호화를 설정하고
이 결과를 리턴한다.

이러한 처리를 하고 실행하면, 로그인과 회원가입 등 모두가 갈 수 있는 페이지를 제외하고,
어떤 페이지로 이동하더라도 로그인 페이지로 우선 이동하게 된다. 로그인이 완료되면 이전에 있던
페이지로 다시 이동하게 된다.
