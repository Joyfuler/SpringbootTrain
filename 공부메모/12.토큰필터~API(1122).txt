리프레시 토큰
--> 처음 접속할 때는 비교적 만료 시간이 짧은 토큰을 발행하고, 
이후 재차 요청이 있는 경우에는 지속 시간이 긴 리프레시 토큰을 부여하여 보안상의 약점을 방지.

요청이 오면 요청의 헤더값을 확인하여 토큰을 식별하고, 유효한 토큰이라면 '콘텍스트 홀더' 라는 곳에
인증 정보를 저장한다.

시큐리티 컨텍스트? -- 인증 객체가 저장되는 보관소. 인증 정보 필요시 필요할 때 꺼내 사용할 수 있음.

request 클래스 / response 클래스 ?
토큰 생성 요청 및 응답을 담당할 dto가 된다.
실습에서는 /api/token으로 POST 요청이 오면 새로운 액세스 토큰을 만든다. (controller)
testUser 객체를 빌더형으로 만들어 정보를 입력. 리프레시 토큰을 만든 후 dto에 testUser의 이메일을 저장한다.
